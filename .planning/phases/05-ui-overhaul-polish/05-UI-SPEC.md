## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, pill badge inline padding (`padding: 2px 8px`) |
| sm | 8px | Compact element spacing, chart legend gaps |
| md | 16px | Default card padding, metric card gaps |
| lg | 24px | Section padding, chart margin |
| xl | 32px | Layout section gaps (heatmap → sector row → news) |
| 2xl | 48px | Body `padding-bottom` to clear fixed ticker strip |
| 3xl | 64px | Page-level top padding |

Exceptions:
- Fixed bottom ticker strip height: 48px (matches `padding-bottom: 48px` on page body)
- Touch targets for settings gear icon: 44px minimum (accessibility)
- LinearProgress bar: 2px height (intentional design, not a spacing token)

Source: CONTEXT.md D-03, D-12, D-21. Defaults applied where not specified.

## Typography

| Role | Size | Weight | Line Height | Notes |
|------|------|--------|-------------|-------|
| Body | 16px | 400 | 1.5 | News feed text, narrative body |
| Label | 12px | 700 | 1.2 | Section labels — uppercase, letter-spacing 0.08em |
| Heading | 20px | 700 | 1.2 | Card headings, chart titles, section headings |
| Display | 28px | 700 | 1.1 | Company price (`current-price`), company name H1 |

Two weights only: 400 (body readability) and 700 (display/label/heading hierarchy). The size difference between 12px Label and 20px Heading provides sufficient visual differentiation without requiring a third weight.

Numeric rendering: all price and percent-change values use `font-variant-numeric: tabular-nums; font-family: 'JetBrains Mono', monospace`.

Source: CONTEXT.md D-24 (section labels spec), REQUIREMENTS.md UI-09. CompanyPage.css already uses `JetBrains Mono` for `.current-price` (2.2rem) and `.metric-card strong` (1.5rem) — new scale anchors to these.

## Interaction Contracts

### Heatmap (UI-01, UI-02)

- Render: Recharts `<Treemap>` with `content` render prop
- Cell sizing: proportional to `market_cap` from `tickers.py` `TICKER_DATA`
- Cell color: linear interpolation across 5-stop diverging palette based on current sentiment score
- Sector grouping: GICS sector nodes wrap constituent stocks; sectors with `stock_count < 3` merged under "OTHER" node
- Sector label: visible overlay inside sector region; font-size 12px, weight 700, uppercase, letter-spacing 0.08em
- "OTHER" label color: `#64748b` (muted)
- Animation: heatmap data reference updated in-place on polling cycle — Recharts re-renders cells without re-animating from zero
- Hover tooltip: ticker symbol, company name, current price, sentiment score (formatted to 2 decimal places), percent change pill badge

### Dual-Axis Chart (UI-03)

- Container: Recharts `<ComposedChart>` with `syncId` for potential future cross-chart sync
- Left Y-axis: price as `<Area>` — stroke `#3b82f6`, fill `rgba(59,130,246,0.1)`, no animation on data update
- Right Y-axis: per-day sentiment as `<Bar>` — domain fixed `[-1, 1]`, per-bar `<Cell>` fill: positive days `#4ade80`, negative days `#f87171`
- X-axis: date labels, 7-day default window
- Chart stays mounted during background refresh — only `data` prop is updated
- Remains mounted during pending narrative state

### Auto-Refresh (UI-04, UI-05)

- Hook: `useInterval` with `setInterval`; clears on unmount
- Tab pause: `document.addEventListener('visibilitychange', ...)` — interval skips tick when `document.visibilityState === 'hidden'`
- In-flight guard: if a refresh is already in-flight, skip the new tick (do not queue a second fetch)
- Refresh indicator: MUI `<LinearProgress>` at `position: fixed; top: 0; left: 0; width: 100%; height: 2px; z-index: 1100`
- Timestamp: "Last updated HH:MM" in 12-hour format; swaps to "Updating..." during refresh cycle
- Interval options: 5 min / 10 min / 30 min; default 10 min; persisted to `localStorage` key `sentiment_refresh_interval`

### Skeleton Loaders (UI-06)

- Trigger: when `loading === true` from `StockDataContext`
- All data-dependent components show `<Skeleton animation="wave">` at exact content height
- Heatmap skeleton: single rectangular block at `height: 400px; width: 100%; border-radius: 16px`
- Chart skeleton: `height: 300px; width: 100%; border-radius: 16px`
- Metric card skeleton: `height: 80px; width: 100%; border-radius: 8px` (one per card)
- News item skeleton: `height: 120px; width: 100%; border-radius: 8px` (repeat 5)
- Narrative skeleton: `height: 96px; width: 100%; border-radius: 8px`
- No layout shift on data arrival — skeleton height matches content height exactly

### Error States (UI-07)

- Scope: per-component inline error, not full-page error overlay
- Layout: centered text block inside the component's normal space
- Contents: (1) which endpoint failed (human-readable name), (2) brief description, (3) "Retry" button (`<Button variant="outlined" size="small">Retry</Button>`)
- Retry: re-triggers the fetch for that component only
- Never shows a blank screen or infinite spinner

### Fixed Bottom Ticker Strip (D-03, D-04)

- Position: `position: fixed; bottom: 0; width: 100%; z-index: 1000; height: 48px`
- Content: ticker symbol, current price, percent change pill badge — all 100 stocks in auto-scrolling marquee
- Animation: CSS `scrollRightToLeft` keyframe (existing from `StockChart.css`) — preserved and updated for dark theme
- Background: `rgba(0,0,0,0.9)` with `backdrop-filter: blur(8px)`
- Body clearance: `padding-bottom: 48px` on page wrapper prevents content from being obscured

### Settings Modal (D-12)

- Trigger: gear icon (`<SettingsIcon>`) in TopBar, right side; `aria-label="Open settings"` declared on the `<IconButton>` wrapper
- Component: MUI `<Dialog>` — `maxWidth="xs"`, `fullWidth`
- Contents: `<Select>` for refresh interval (5 min / 10 min / 30 min), "Save Settings" button, "Keep Current Settings" button
- Save Settings: updates React state and writes to `localStorage`
- Keep Current Settings: closes dialog without applying changes
- Design: extensible — additional settings rows can be added without layout changes

### Percent Change Pill Badge (UI-10, D-23)

- Markup: `<span>` with class `pct-badge`
- Positive: `background: rgba(22,163,74,0.15); color: #4ade80; border-radius: 9999px; padding: 2px 8px; font-variant-numeric: tabular-nums; font-family: 'JetBrains Mono', monospace`
- Negative: `background: rgba(220,38,38,0.15); color: #f87171` — same structure
- Applies everywhere percent change is displayed: ticker strip, metric cards, heatmap tooltip, company header

### AI Narrative (D-18, D-19)

- Pending state: `<Skeleton animation="wave" height={96}>` with caption "Generating narrative..."
- Loaded state: narrative text + muted caption `"Generated X min ago"` (computed from `generated_at` ISO string)
- Caption style: `font-size: 12px; color: #94a3b8; margin-top: 8px`
- Polling: re-poll `GET /stock-narrative/{ticker}` every 10 seconds while `status === "pending"`; stop when `status === "complete"` or after 5 minutes (timeout)

## Claude's Discretion Decisions

These were left open in CONTEXT.md and are resolved here:

| Question | Decision |
|----------|----------|
| Recharts Treemap `content` render prop | `CustomTreemapContent` component — receives `x, y, width, height, name, sentiment, sector`; renders `<rect>` with interpolated fill, `<text>` for ticker symbol (visible when cell width > 40px), sector label rendered in sector root node |
| StockDataContext shape | `{ stocks, loading, error, lastUpdated, isRefreshing, refresh }` — `loading` is true only on first load; `isRefreshing` is true on subsequent background refreshes |
| Heatmap tooltip | Show on hover: ticker symbol (bold, 14px), company name (12px, muted), current price (JetBrains Mono), sentiment score (formatted `+0.42`), percent change pill badge |
| In-flight refresh guard | If `isRefreshing === true` when tick fires, skip the tick silently (no queuing) |
| Tablet breakpoint | 768px (matches existing CompanyPage.css `@media (max-width: 768px)`) |
