# Requirements: RealTime Stock Sentiment Analysis Engine

**Defined:** 2026-03-26
**Core Value:** A recruiter or engineer who opens this app immediately sees what makes stocks move — sentiment + price in one view — before reading a single line of code.

---

## v1 Requirements

### Security

- [ ] **SEC-01**: Hardcoded AWS credentials removed from source and git history; all secrets loaded via environment variables
- [ ] **SEC-02**: CORS restricted to specific frontend origin(s) — wildcard `*` with credentials removed
- [ ] **SEC-03**: All API endpoints protected by API key header (`X-API-Key`)
- [ ] **SEC-04**: `/analyze-custom` text input validated with max length (2000 chars) via Pydantic `Query`
- [ ] **SEC-05**: `.dockerignore` added to prevent `.env` files being baked into Docker image layers
- [ ] **SEC-06**: `team-provider-info.json` and other AWS config files added to `.gitignore`

### Backend Performance

- [ ] **PERF-01**: ML models (FinBERT + Qwen2.5) loaded via FastAPI `lifespan` context manager into `app.state` — no blocking at import time
- [ ] **PERF-02**: Model warm-up inference call executed before server accepts traffic to eliminate first-request latency spike
- [ ] **PERF-03**: yfinance calls wrapped in `run_in_executor` and fanned out concurrently with `asyncio.gather` + semaphore (max 10 concurrent)
- [ ] **PERF-04**: In-memory cache replaced with `cachetools.TTLCache` — 900s TTL for stock history, 300s for news
- [ ] **PERF-05**: Qwen inference offloaded to `asyncio.Queue` background worker; `/analyze-custom` returns `job_id` immediately; `GET /analyze-custom/{job_id}` for polling

### Data Pipeline

- [ ] **DATA-01**: Ticker list expanded from 15 to S&P 100 (102 tickers) organized by GICS sector, hardcoded in `tickers.py`
- [ ] **DATA-02**: yfinance history fetched via `yf.download()` in chunks of 50 (2 requests instead of 100 sequential)
- [ ] **DATA-03**: News fetching uses tiered priority — Tier 1 (top 20 by market cap, always fetched), Tier 2 (next 40, rotating), Tier 3 (remaining, on-demand)
- [ ] **DATA-04**: News deduplication by UUID to prevent duplicate articles across tickers

### Sentiment Intelligence

- [x] **SENT-01**: FinBERT scoring upgraded to full-probability formula: `score = P(positive) - P(negative)` using `AutoModelForSequenceClassification` (replaces pipeline shortcut)
- [x] **SENT-02**: Article-level scores aggregated to daily stock-level score using confidence-weighted mean: `Σ(score_i × conf_i) / Σ(conf_i)`
- [x] **SENT-03**: Sentiment trend time series built per stock using EMA (span=5 for 7-day, span=20 for 30-day) via pandas `ewm()`
- [x] **SENT-04**: Sector-level sentiment computed as equal-weight average of constituent stock scores (displayed only when `stock_count >= 3`)
- [x] **SENT-05**: Per-stock AI narrative generated by Qwen: concise "why is this stock moving" summary from top 8 headlines + their pre-computed scores

### UI — Core Dashboard

- [ ] **UI-01**: Sentiment heatmap using Recharts `<Treemap>` — all 100 stocks as cells, sized by market cap, colored by sentiment score (5-stop diverging palette: `#16a34a` → `#4ade80` → `#475569` → `#f87171` → `#dc2626`)
- [ ] **UI-02**: Stocks grouped by GICS sector on the heatmap with sector label overlay (only when `stock_count >= 3`)
- [x] **UI-03**: Price + sentiment dual-axis chart using Recharts `<ComposedChart>` — `<Area>` for price (left Y), `<Bar>` with per-cell `<Cell>` coloring for sentiment (right Y, domain `-1` to `1`)
- [x] **UI-04**: Auto-refresh with configurable polling interval (default 10 min); `useInterval` hook pattern; chart stays rendered during refresh (no unmount/remount)
- [x] **UI-05**: Last-updated timestamp visible in UI, swaps to "Updating…" during refresh cycle

### UI — Polish & Quality

- [ ] **UI-06**: Skeleton loaders for all data-dependent components (metric cards, charts, heatmap, news feed) — MUI `<Skeleton animation="wave">` with dark-theme `bgcolor` override
- [ ] **UI-07**: Informative error states (not blank screens) for all fetch failure scenarios
- [x] **UI-08**: Responsive layout — usable on tablet and desktop; stock detail pages scroll gracefully on narrow viewports
- [ ] **UI-09**: Typography upgrade — section labels in all-caps with letter-spacing; price/change numbers use `font-variant-numeric: tabular-nums`
- [ ] **UI-10**: Percent change displayed as color-tinted pill badge (not just colored text)

### Cleanup

- [ ] **CLEAN-01**: Stale AWS API Gateway hardcoded URLs removed from `src/apis/api.js` and `src/utils/getStockData.js`; `REACT_APP_API_URL` required at build time
- [ ] **CLEAN-02**: MUI v4 (`@material-ui/core`) removed; consolidated to MUI v5 only — eliminates `--legacy-peer-deps` requirement
- [ ] **CLEAN-03**: Unused `aws-amplify` and `@aws-amplify/ui-react` dependencies removed from frontend
- [ ] **CLEAN-04**: `test_output.json` added to `.gitignore` (untracked file with potential API response data)

## v2 Requirements

### Enhanced Analytics

- **V2-01**: Watchlist — user can pin specific tickers for persistent tracking
- **V2-02**: Sentiment alerts — notification when a stock's sentiment shifts by >0.3 in 24h
- **V2-03**: Correlation view — scatter plot of sentiment score vs. next-day price change
- **V2-04**: Historical sentiment search — query sentiment for any past date range

### Infrastructure

- **V2-01**: CI/CD pipeline (GitHub Actions) — automated Docker build + test on PR
- **V2-02**: Persistent storage (SQLite or Redis) — replace in-memory cache with durable store
- **V2-03**: WebSocket streaming — push updates to frontend instead of polling

## Out of Scope

| Feature | Reason |
|---------|--------|
| Open-ended ticker search | Requires search infra; top 100 is the right portfolio boundary |
| User accounts / auth | Adds complexity without portfolio value |
| Trading / order execution | Purely a research tool; trading adds regulatory and complexity concerns |
| Mobile native app | Responsive web is sufficient |
| Paid data sources | Free yfinance + Yahoo Finance news sufficient for portfolio |
| Real-time WebSocket streaming | TTL polling is sufficient; WebSockets deferred to v2 |
